## Detailed Analysis

### 1. Performance Issues

#### P0: Memory Leak - Multiple Intervals
**Location:** `App.jsx` lines 30-56

**Problem:**
```javascript
useEffect(() => {
  const fetchQueue = async () => { /* ... */ };
  fetchQueue();
  const id = setInterval(fetchQueue, POLL_INTERVAL);
  return () => clearInterval(id); // ❌ Only cleans up ONE interval
}, []);

useEffect(() => {
  const fetchTasks = async () => { /* ... */ };
  fetchTasks();
  const id = setInterval(fetchTasks, POLL_INTERVAL);
  return () => clearInterval(id);
}, []);
```

**Impact:** Every tab change creates 2 new intervals. After 10 tab switches: 20 intervals polling every 3 seconds = performance death.

**Fix:**
```javascript
useEffect(() => {
  const fetchAll = async () => {
    await Promise.all([getQueueStatus(), getTasks()]);
  };
  
  fetchAll();
  const id = setInterval(fetchAll, POLL_INTERVAL);
  return () => clearInterval(id); // ✅ Single interval, cleanup works
}, []);
```

---

#### P1: Polling When Tab Is Inactive
**Location:** `App.jsx` line 19

**Problem:** Polls every 3 seconds even when user switched tabs or closed the window.

**Fix:** Use Page Visibility API
```javascript
useEffect(() => {
  const fetchAll = async () => { /* ... */ };
  
  fetchAll();
  const id = setInterval(fetchAll, POLL_INTERVAL);
  
  // Stop polling when tab hidden
  const handleVisibility = () => {
    if (document.hidden) {
      clearInterval(id);
    } else {
      fetchAll();
      id = setInterval(fetchAll, POLL_INTERVAL);
    }
  };
  
  document.addEventListener('visibilitychange', handleVisibility);
  return () => {
    clearInterval(id);
    document.removeEventListener('visibilitychange', handleVisibility);
  };
}, []);
```

### 2. UX Issues

#### P0: No User Feedback on Actions
**Location:** `App.jsx` lines 74-139

**Problem:** User clicks "Create Task" → nothing visible happens for 2 seconds → user clicks again → creates 5 duplicate tasks.

**Current:**
```javascript
const handleCreateTask = async (e) => {
  e.preventDefault();
  setLoading(true);
  await createTask(newTask); // ← 2 seconds of silence
  setLoading(false);
};
```

**Better:**
```javascript
const handleCreateTask = async (e) => {
  e.preventDefault();
  setCreating(true);
  showToast('Creating task...', 'info'); // ← Immediate feedback
  try {
    await createTask(newTask);
    showToast('Task created!', 'success');
  } catch (error) {
    showToast('Failed: ' + error.message, 'error');
  } finally {
    setCreating(false);
  }
};
```

### 5. State Management

#### P1: Race Condition Risk
**Location:** Lines 22-28

**Problem:** Multiple `useEffect` hooks updating state independently can cause stale closures.

**Current:**
```javascript
const [queue, setQueue] = useState({...});
const [tasks, setTasks] = useState([]);
const [settings, setSettings] = useState({...});
```

Each effect updates independently → can overwrite each other.

**Better:**
```javascript
const [state, dispatch] = useReducer(reducer, initialState);
// Atomic updates, no race conditions
```
